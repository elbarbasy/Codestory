# 08. Portfolio Page

## Objective
Showcase CodeStory's best work, demonstrate technical capability, and convert visitors into qualified leads.

## Page Structure
1. Hero
2. Portfolio Statistics
3. Category Filter
4. Project Grid
5. Case Study Preview
6. Development Technologies
7. CTA

## Hero
Headline: "Our Work Speaks for Itself"
Subtitle: Display selected projects built for businesses, startups, and organizations.

## Statistics
- Total Projects
- Happy Clients
- Technologies Used
- Average Delivery Time

## Category Filter
Provide filters without page reload:
- All
- Company Profile
- Landing Page
- E-Commerce
- Web App
- UI/UX

## Project Card
Each project card should include:
- Thumbnail image
- Project name
- Category
- Short description
- Technology badges
- Completion year
- Live Demo button
- Source Code button (optional)

## Hover Interaction
- Slight image zoom
- Card elevation
- Blue border glow
- Action buttons fade in

## Project Detail Modal or Page
Include:
- Overview
- Client goals
- Challenges
- Solution
- Technology stack
- Features delivered
- Gallery
- Testimonial (if available)

## Technologies Section
Highlight commonly used technologies:
- Next.js
- Laravel
- React
- Tailwind CSS
- PostgreSQL
- Supabase
- Vercel

## Empty State
If no projects match a filter, show a friendly message and suggest viewing all projects.

## Final CTA
Headline: "Let's Build Your Next Project"
Buttons:
- Konsultasi Gratis
- WhatsApp

## Acceptance Criteria
The portfolio should be visually impressive, fast to browse, responsive on all devices, and clearly demonstrate CodeStory's expertise.
