# 17. Responsive Rules

## Objective
Ensure the CodeStory website feels polished and usable on every screen size.

## Breakpoints
Use a simple responsive system:
- Mobile: small screens
- Tablet: medium screens
- Desktop: large screens
- Wide desktop: extra large screens

## Mobile Rules
- Use a hamburger menu
- Keep the language toggle beside the hamburger icon
- Stack most layout sections vertically
- Increase tap targets
- Reduce decorative clutter
- Keep content readable without zooming

## Tablet Rules
- Use mixed layouts where appropriate
- Keep cards balanced and not overly wide
- Preserve strong spacing

## Desktop Rules
- Show full navbar
- Use multi-column layouts
- Use richer motion and hover interactions
- Keep content centered with comfortable max width

## Wide Desktop Rules
- Prevent content from stretching too far
- Maintain a readable line length
- Keep hero and mockups visually balanced

## Component Behavior
### Navbar
- Desktop: full menu
- Mobile: hamburger + language toggle

### Pricing Cards
- Stack on mobile
- Grid on desktop

### Portfolio Cards
- Single column or horizontal list on mobile if needed
- Multi-column grid on desktop

### Forms
- Single column on mobile
- Wider layout on desktop if helpful

## Typography Rules
- Headings should scale down elegantly on smaller screens
- Body text must stay readable without crowding

## Motion Rules
- Reduce or simplify motion on low-power devices when necessary
- Respect reduced-motion preferences

## Acceptance Criteria
Every page should remain easy to navigate and visually consistent from mobile phones to large desktop monitors.
